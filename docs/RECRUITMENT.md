# نظام إدارة دورة التعيين (Recruitment Pipeline)

دليل تشغيل واختبار نظام المرشحين في الداشبورد.

## 1. التهيئة الأولى

1. تأكد من متغيرات Google Sheets في `.env.local` (نفس إعدادات الداشبورد الحالي).
2. شغّل مرة واحدة:

```bash
npx tsx scripts/ensure-recruitment-sheets.ts
```

ينشئ التبويبات: `مرشحين_التعيين`، `سجل_نشاط_المرشحين`، `إشعارات_التعيين`.

## 2. إنشاء حساب «مسؤول التعيينات»

1. افتح Google Spreadsheet الرئيسي → تبويب **Admins**.
2. أضف صفاً جديداً:

| الكود | الاسم | كلمة المرور | الصلاحيات |
|-------|-------|-------------|-----------|
| REC001 | مسؤول تعيينات | your_password | `recruitment_manager` |

> **مهم:** عمود الصلاحيات يجب أن يكون بالضبط `recruitment_manager` (بدون `limited:`).

3. من صفحة الدخول اختر **مسؤول التعيينات** وأدخل الكود وكلمة المرور.
4. ستُوجَّه إلى `/recruitment` ولن ترى صفحات `/admin`.

## 3. الأدمن الكامل

- الأدمن بدون `limited:` يرى قسم التعيين في القائمة الجانبية + إحصائيات في لوحة الأدمن.
- أدمن محدود: أضف `recruitment` لقائمة الصلاحيات، مثال:
  `limited:dashboard,recruitment,riders`

## 4. بيانات تجريبية (Seed)

```bash
npx tsx scripts/seed-recruitment-data.ts
```

## 5. المسارات

| المسار | الوصف |
|--------|--------|
| `/recruitment` | لوحة إحصائيات |
| `/recruitment/candidates` | جميع المتقدمين النشطين |
| `/recruitment/archive` | الأرشيف وإعادة التفعيل |
| `/recruitment/bulk-import` | استيراد Excel/CSV أو نص |

## 6. اختبار عدم تأثر النظام القديم

بعد التحديث، تحقق يدوياً:

- [ ] تسجيل دخول **مشرف** → `/dashboard`، المناديب، المعدات
- [ ] تسجيل دخول **مدير** → `/admin/dashboard`، الرواتب، طلبات التعيين القديمة (`/admin/assignment-requests`)
- [ ] تسجيل دخول **مسؤول تعيينات** → `/recruitment` فقط؛ محاولة فتح `/admin` تُعيد التوجيه

## 7. ملاحظات

- **طلبات التعيين** (`طلبات_التعيين`) = ربط المناديب بالمشرفين (النظام القديم).
- **مرشحين التعيين** (`مرشحين_التعيين`) = pipeline التوظيف الجديد (منفصل تماماً).
- سجل النشاطات يظهر للأدمن فقط من زر «سجل» في الجدول.
- الإشعارات تُحدَّث تلقائياً كل دقيقتين في الشريط الجانبي.
