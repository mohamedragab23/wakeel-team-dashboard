# نظام التعليقات اليومية للمناديب وتدقيق البيانات

## نظرة عامة
تم إضافة 3 أنظمة جديدة لتحسين دقة التحليلات والتقارير:

1. **نظام التعليقات اليومية** (`rider_daily_comments`)
2. **تصدير المناديب الأشباح** (`ghost_riders_export`)
3. **تدقيق البيانات الناقصة** (`missing_data_audit`)
4. **تصحيح Code Normalization** (تمييز كود المندوب vs المشرف)

---

## 1️⃣ نظام التعليقات اليومية للمناديب

### الهدف
تسجيل حالة كل مندوب يومياً مع السبب المحدد (حادث، إجازة مرضية، عطل معدات، إلخ) لتحليل أكثر دقة وفهم الأسباب الحقيقية وراء غياب أو ضعف أداء المناديب.

### المكونات

#### **الصفحة الرئيسية**: `/rider-comments`
- **الوصول**: متاح للمشرفين والإدارة
- **الوظائف**:
  - إضافة تعليق جديد لمندوب محدد في تاريخ محدد
  - اختيار الفئة من قائمة منسدلة (حادث، إجازة مرضية، إلخ)
  - إضافة تاريخ العودة المتوقع (للحوادث/الإجازات المرضية)
  - إضافة ملاحظات نصية إضافية
  - عرض التعليقات الأخيرة (آخر 20 تعليق)

#### **الفئات المتاحة**:
```typescript
type CommentCategory =
  | 'accident'            // 🚑 حادث
  | 'medical_leave'       // 🏥 إجازة مرضية
  | 'family_emergency'    // 👨‍👩‍👧 حالة عائلية طارئة
  | 'equipment_issue'     // 🛠️ عطل معدات
  | 'frequent_absences'   // ⚠️ غياب متكرر
  | 'vacation'            // 🏖️ إجازة
  | 'poor_performance'    // 📉 ضعف أداء
  | 'terminated'          // ⛔ إقالة
  | 'other';              // 📝 أخرى
```

#### **Google Sheet**: `rider_daily_comments`
- **الأعمدة**:
  - `id` (UUID)
  - `riderCode` (كود المندوب)
  - `riderName` (اسم المندوب)
  - `supervisorCode` (كود المشرف)
  - `supervisorName` (اسم المشرف)
  - `date` (التاريخ بصيغة YYYY-MM-DD)
  - `category` (الفئة)
  - `expectedReturnDate` (تاريخ العودة المتوقع - اختياري)
  - `estimatedReturnDays` (عدد الأيام المتوقعة - اختياري)
  - `notes` (ملاحظات نصية)
  - `createdAt` (تاريخ الإنشاء ISO)
  - `updatedAt` (تاريخ التحديث ISO)

#### **API Routes**:
- `GET /api/rider-comments`
  - Query params: `riderCode`, `supervisorCode`, `startDate`, `endDate`
  - يدعم تصفية المشرف (يعرض تعليقات مناديبه فقط)
  - الإدارة يمكنهم رؤية جميع التعليقات

- `POST /api/rider-comments`
  - Body: `{ riderCode, riderName, date, category, expectedReturnDate?, estimatedReturnDays?, notes }`
  - يتطلب JWT authentication (supervisor/admin)

#### **Service Functions** (`lib/riderComments/service.ts`):
```typescript
getRiderComments(riderCode, startDate, endDate)
getCommentsForDate(date)
getSupervisorComments(supervisorCode, startDate, endDate)
addRiderComment(comment)
getAbsenceReasonsSummary(startDate, endDate, supervisorCode?)
getRidersExpectedToReturn(withinDays)
```

### الفوائد
✅ **تحليل أدق**: معرفة السبب الحقيقي وراء غياب المندوب  
✅ **توقع العودة**: معرفة متى سيعود المناديب المصابون/المرضى  
✅ **تحذيرات مبكرة**: اكتشاف المناديب الذين يأخذون إجازات متكررة  
✅ **قرارات أفضل**: توصيات مبنية على أسباب حقيقية وليس مجرد أرقام  
🔒 **سجل دائم**: التعليقات لا تُحذف، مما يوفر سجلاً كاملاً لحالة كل مندوب

## 2️⃣ تصدير المناديب الأشباح

### الهدف
تحديد المناديب الذين لديهم بيانات أداء في الشيت ولكن لا يوجدون في نظام إدارة المناديب (غير مسجلين، أو لم يتم إضافتهم بعد).

### المكونات

#### **الصفحة الرئيسية**: `/admin/ghost-riders-export`
- **الوصول**: الإدارة فقط
- **الوظائف**:
  - اختيار فترة زمنية (تاريخ البداية والنهاية)
  - اختيار المنطقة (اختياري)
  - تصدير البيانات إلى ملف Excel

#### **API Route**:
- `GET /api/admin/ghost-riders-export`
  - Query params: `startDate`, `endDate`, `zone?`, `supervisorCode?`
  - يعيد ملف Excel جاهز للتنزيل

#### **البيانات المُصدَّرة**:
- كود المندوب (في شيت الأداء)
- اسم المندوب
- عدد الأيام في الفترة
- إجمالي الساعات
- إجمالي الطلبات
- متوسط الساعات اليومية
- هل موجود في نظام المناديب؟ (Yes/No)

### الفوائد
✅ معرفة المناديب الذين يجب تسجيلهم  
✅ اكتشاف أخطاء الإدخال في شيت الأداء  
✅ تحسين دقة التقارير الاستراتيجية  

---

## 3️⃣ تدقيق البيانات الناقصة

### الهدف
عرض جميع المناديب الذين لديهم بيانات ناقصة (مشرف، منطقة، تاريخ انضمام) مع حساب نسبة اكتمال البيانات.

### المكونات

#### **الصفحة الرئيسية**: `/admin/missing-data-audit`
- **الوصول**: الإدارة فقط
- **الإحصائيات المعروضة**:
  - إجمالي المناديب
  - نسبة اكتمال البيانات (%)
  - عدد المناديب الذين لديهم مشاكل
  - الحالة (ممتاز/يحتاج تحسين/حرج)

#### **توزيع المشاكل**:
- عدد المناديب بدون مشرف
- عدد المناديب بدون منطقة
- عدد المناديب بدون تاريخ انضمام
- عدد المناديب بدون اسم

#### **جدول المناديب**:
- يعرض جميع المناديب الذين لديهم بيانات ناقصة
- فلترة حسب نوع المشكلة
- عرض تفصيلي لكل مندوب (الكود، الاسم، المشرف، المنطقة، تاريخ الانضمام، المشاكل)

#### **API Route**:
- `GET /api/admin/missing-data-audit`
  - يعيد: `{ summary: {...}, riders: [...] }`

### التحذيرات
⚠️ إذا كانت نسبة الاكتمال أقل من 80%:
- التقارير في "مركز العمليات الاستراتيجي" قد لا تكون دقيقة
- بعض التحليلات الاستراتيجية معطلة حتى تتحسن جودة البيانات
- يُنصح بإكمال البيانات الناقصة في أقرب وقت ممكن

### الفوائد
✅ **رؤية واضحة** لجودة البيانات  
✅ **تحسين دقة التحليلات** الاستراتيجية  
✅ **إرشادات واضحة** لكيفية إصلاح المشاكل

## 4️⃣ تصحيح Code Normalization

### المشكلة السابقة
كانت دالة `normalizeRiderCodeForPerformance` تتعامل مع جميع الأكواد على أنها أرقام فقط، مما كان يؤدي إلى:
- حذف البادئة `WA-` من كود المشرف
- خلط بين كود المندوب وكود المشرف
- أخطاء في مطابقة البيانات

### التصحيح
**كود المندوب**: أرقام فقط (مثل: `877614`, `2520149`)  
**كود المشرف**: يبدأ بـ `WA-` (مثل: `WA-001`, `WA-016`)

#### **الدوال الجديدة** (`lib/riderCodeUtils.ts`):
```typescript
// تحقق: هل الكود كود مشرف؟
isSupervisorCode(code: unknown): boolean

// تحقق: هل الكود كود مندوب؟
isRiderCode(code: unknown): boolean

// تطبيع الكود (يحافظ على كود المشرف كما هو)
normalizeRiderCodeForPerformance(code: unknown): string
```

#### **السلوك الجديد**:
- إذا كان الكود يبدأ بـ `WA-` → يُعاد كما هو (uppercase, trimmed)
- إذا كان الكود رقمي → يُحذف الأصفار البادئة
- أي شيء آخر → يُجرى عليه تطبيع رقمي

### الفوائد
✅ **تمييز دقيق** بين كود المندوب والمشرف  
✅ **مطابقة صحيحة** في جميع التقارير  
✅ **عدم حذف معلومات** كود المشرف

## خطوات النشر

### 1. إعداد Google Sheet
تأكد من وجود شيت بعنوان `rider_daily_comments` في نفس الملف الرئيسي، أو سيتم إنشاؤه تلقائياً عند أول تعليق.

**الأعمدة**:
```
id | riderCode | riderName | supervisorCode | supervisorName | date | category | expectedReturnDate | estimatedReturnDays | notes | createdAt | updatedAt
```

### 2. صلاحيات الإدارة
إذا كان لديك admins بصلاحيات محدودة (`limited:`), تأكد من إضافة:
```
limited:...,ghost_riders_export,rider_comments,missing_data_audit
```

### 3. اختبار الميزات
1. **التعليقات اليومية**:
   - افتح `/rider-comments`
   - اختر مندوب من القائمة
   - أضف تعليق بفئة محددة
   - تحقق من ظهور التعليق في القائمة الجانبية

2. **تصدير الأشباح**:
   - افتح `/admin/ghost-riders-export`
   - حدد فترة زمنية ومنطقة
   - قم بتصدير البيانات وافتح ملف Excel

3. **تدقيق البيانات**:
   - افتح `/admin/missing-data-audit`
   - تحقق من نسبة الاكتمال
   - استعرض الجدول وفلتر حسب المشكلة

4. **Code Normalization**:
   - تحقق من أن تقارير المشرفين والمناديب تعمل بشكل صحيح
   - تأكد من عدم اختلاط البيانات بين المشرفين والمناديب

## الأسئلة الشائعة

### س: هل التعليقات تُحذف؟
**ج**: لا، التعليقات دائمة ولا تُحذف. هذا يوفر سجلاً تاريخياً كاملاً لحالة كل مندوب.

### س: من يمكنه رؤية التعليقات؟
**ج**: 
- **المشرفون**: يرون تعليقات مناديبهم فقط
- **الإدارة**: يرون جميع التعليقات

### س: ماذا لو كانت نسبة اكتمال البيانات منخفضة؟
**ج**: يجب إكمال البيانات الناقصة في أقرب وقت ممكن. نسبة أقل من 80% تعني أن التقارير الاستراتيجية قد لا تكون دقيقة.

### س: كيف أستخدم بيانات الأشباح؟
**ج**: استخدم ملف Excel المُصدَّر لمعرفة المناديب الذين يجب تسجيلهم في نظام إدارة المناديب. ثم قم بإضافتهم يدوياً أو تصحيح أكوادهم في شيت الأداء.
